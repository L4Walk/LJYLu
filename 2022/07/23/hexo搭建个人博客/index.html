利用hexo搭建个人博客

发表于2022-07-23|更新于2022-07-26|折腾博客
|阅读量:

前言

typecho主题用的不习惯，想新折腾，于是来搞搞Hexo。

主要步骤

  • 安装git
  • 安装Node.js
  • 安装Hexo
  • 给Hexo换肤
  • 部署Hexo到GitHub

安装 git

主要过程略。

Git下载地址：https://git-scm.com/download/

安装完成后，可以右键 git bash here

输入git --vesion，可以看到git的版本号

下载慢可以使用淘宝镜像源地址：https://npm.taobao.org/mirrors/git-for-windows/

安装 Node.Js

主要过程略

安装地址：https://nodejs.org/en/download/

镜像源：https://npm.taobao.org/mirrors/node

右键git bash here
输入node -v，可查看当前node版本。

安装 Hexo

安装

安装完必备的应用程序后，使用 npm 安装 Hexo。

选择需要的盘，新建一个文件夹，我这里创建了“MyBlog”，进入“MyBlog”（这个文件夹就是博客的根目录），右键空白处，点击git bash here。

输入npm install -g hexo-cli

输入hexo -v查看hexo版本

初始化博客站点

输入hexo init，初始化博客站点。

看到INFO Start blogging with Hexo! 即成功安装。

之后输入npm install，安装相关依赖。

之后查看文件夹，会有如下这些东西：

以后发布文章什么的，都在这个根目录下操作，可以直接在这里右键git bash here。

输入hexo s

可以看到博客已经在本地搭建成功了。

但是初始界面不是特别好看，所以可以优化一下它。

美化博客

这里推荐使用Butterfly主题。

下载 Butterfly

在根目录下，右键git bash here。
输入：git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

在theme文件夹下，可以看到主题已经安装了

另外这个主题需要一些插件，如果没有pug 以及 stylus 的渲染器，输入npm install hexo-renderer-pug hexo-renderer-stylus --save 安装

升级建议

为了减少以后升级主题带来的不便，可以采用官方推荐的方法：

在博客的根目录创建一个文件_config.butterfly.yml，并把主题目录的_config.yml内容复制到_config.butterfly.yml去。（注意：复制的是主题的_config.yml，而不是hexo根目录下的_config.yml）。

注意：不要把主题目录的_config.yml删掉

注意：以后只需要在_config.butterfly.yml进行配置就行。
如果使用了_config.butterfly.yml，则配置主题中的_config.yml将不会有效果

应用 Butterfly

在根目录下，找到_config.yml文件

找到theme

改成与主题文件夹相应的名字，此处即butterfly

之后依次输入 hexo clean，hexo g，hexo s。

常用命令 简写 作用
hexo clean 清楚缓存和已经生成的静态文件
hexo generate hexo g 生成静态文件
hexo server hexo s 本地启动
hexo deploy hexo d 部署网站

一般情况都是依次输入 hexo clean，hexo g，hexo s。没问题再hexo d即可。

让我们打开本地博客文件，可以看到主题已经布置成功了。

部署到 Github

创建一个博客仓库，注意仓库名和Github账户名称相同。

复制这个网址

打开根目录下的_config.yml，在Deployment下修改：
type: git
repo: 你的Github网址
branch: master

然后想把电脑上的文件推送到github上建立的仓库，需要安装hexo-deployer-git，在根目录下右键git bash here。
输入：
npm install hexo-deployer-git --save

输入hexo d

之后根据提示即可安装，显示INFO Deploy done: git就代表成功了。

刷新Github仓库页面，可以发现已经上传成功了。

打开Settings，在侧面栏中选中Pages，如下设置并点击Save：

可以看到域名是 https://ljylu.github.io/LJYLu/

页面加载不正常

打开后页面发现加载不正常

F12发现是js未成功加载，报net::ERR_ABORTED 404

原因是我们并未配置url。
打开根目录下面的_config.yml
在url中把修改url地址，即改为你的Github pages的地址。
注意，结尾一定要有/。

之后依次输入 hexo clean，hexo g，hexo d。

刷新界面，可以看到已经修改成功了

写博客

这里有两种方法写博客

方法一

在博客根目录下git bash here，然后输入hexo new "文章名字"，之后会在_posts文件夹里看到创建的md文件，之后就可以打开编辑

方法二

直接写好，之后以md格式保存到_posts文件夹（建议参考hello-world.md的格式）。

示例

这篇文章是我在本地写好后保存到_posts文件夹里。

之后依次输入hexo g，hexo d，之后就会发现文章更新了。

结语

hexo d之后需要略等几秒才会更新，可以多刷新几次。

之后就要进行以前的数据迁移和新网站建设了。
